# Science Paper Agent

Telegram-бот, который каждый день находит и присылает релевантные научные статьи по профилю исследователя.

Пользователь указывает свои интересы и ключевые слова, а бот сам ищет статьи на arXiv и HuggingFace, оценивает их релевантность с помощью LLM и присылает топ-3 с персонализированными обзорами.

---

## Две модели: зачем и какие

Бот использует **две LLM-модели** через OpenAI-совместимый API:

| Роль | Для чего | Требования | Модели, использованные на тесте |
|------|----------|------------|---------------|
| **Scoring** (оценка) | Быстро оценить релевантность каждой статьи (0-100). Вызывается много раз (по разу на каждую найденную статью). | Быстрая, дешёвая | `gpt-4o-mini` |
| **Generation** (генерация) | Написать развёрнутый обзор для топ-3 отобранных статей. Вызывается 3 раза за дайджест. | Умная, качественный текст | `qwen/qwen3.5-397b-a17b` |

Scoring-модель прогоняет 20-50 статей, поэтому важна скорость и цена. Generation-модель пишет обзоры только для финалистов, поэтому тут важно качество текста.

## Быстрый старт

### 1. Создать Telegram-бота

1. Открой Telegram, найди [@BotFather](https://t.me/BotFather)
2. Напиши `/newbot`
3. Придумай имя (например, "My Paper Agent") и username (например, `my_paper_agent_bot`)
4. BotFather выдаст токен вида `123456789:ABCdefGHIjklMNOpqrsTUVwxyz` — сохрани его

### 2. Получить API-ключ для LLM

Бот работает через любой OpenAI-совместимый API (OpenAI, OpenRouter, Together, и т.д.):

1. Зарегистрируйся у провайдера
2. Получи API-ключ
3. Укажи `LLM_BASE_URL` и `LLM_API_KEY` в настройках (шаг 4)

### 3. Установить зависимости

```bash
python -m venv venv
source venv/bin/activate      # Linux/Mac
# venv\Scripts\activate       # Windows

pip install -r requirements.txt
```

### 4. Настроить переменные окружения

```bash
cp .env.example .env
```

Отредактируй `.env` — впиши свой токен бота и API-ключ:

```bash
# ОБЯЗАТЕЛЬНЫЕ
TELEGRAM_BOT_TOKEN=сюда_токен_от_BotFather
LLM_API_KEY=сюда_ключ_от_провайдера

# ОПЦИОНАЛЬНЫЕ (есть значения по умолчанию)
LLM_BASE_URL=https://api.openai.com/v1
LLM_SCORING_MODEL=gpt-4o-mini
LLM_GENERATION_MODEL=google/gemini-2.5-flash
```

### 5. Запустить

```bash
python run.py
```

Бот запустится и будет ждать сообщений. Напиши ему `/start` в Telegram.

| Команда | Что делает |
|---------|-----------|
| `/start` | Регистрация, онбординг |

| `/help` | Справка |

## Как это работает

1. Пользователь через `/start` указывает интересы и ключевые слова
2. Каждый день в заданное время (по умолчанию 9:00 МСК):
   - Бот ищет свежие статьи на arXiv и HuggingFace по ключевым словам
   - **Scoring-модель** оценивает релевантность каждой статьи (0-100)
   - Статьи дополнительно обогащаются данными о цитированиях (Semantic Scholar, OpenAlex)
   - Итоговый рейтинг: 70% релевантность + 30% авторитетность
   - **Generation-модель** пишет обзор для топ-3 статей
3. Пользователь может оценивать рекомендации (лайк/дизлайк), и бот учитывает это в будущих оценках

## Структура проекта

```
src/
├── main.py            # Точка входа
├── config.py          # Конфигурация из .env
├── bot/               # Telegram-хендлеры и клавиатуры
├── agent/             # Оркестратор: поиск -> оценка -> генерация
├── llm/               # Клиент LLM (OpenAI-совместимый)
├── parsers/           # Парсеры источников (arXiv, HuggingFace, обогащение)
├── db/                # SQLite: модели и репозитории
└── scheduler/         # Планировщик ежедневной рассылки
```

## Тестирование компонентов

Перед полным запуском можно проверить отдельные части:

```bash
python scripts/test_db.py      # Проверка базы данных
python scripts/test_arxiv.py   # Проверка парсера arXiv (без API-ключа)
python scripts/test_llm.py     # Проверка LLM (нужен LLM_API_KEY)
```
